## v2 迭代:模块化顶部导航

**变更原因**:用户表示"抓取网站"只是未来多个功能模块之一,需要在顶部预留模块切换栏。

**方案**:
- 标题栏(52px)下方插入 **Module Tab Bar**(48px,白底,1px 底分割线)
- 当前激活 Tab:`📥 站点资源抓取`(紫色选中态)
- 灰显占位 Tab:`🔄 批量重命名`(将来) `📦 资源整理`(将来) `⚙️ 设置`(将来)
- Body 内容区保持原"URL 输入 → 文件类型 Chip → 存储目录 → 停止/开始"流程不变
- Body 顶部 padding 缩小,让整体仍填满 680 高度

**视觉规则**:
- 激活 tab:紫色品牌色 + 底部 2px 紫色下划线 + 紫色文字
- 灰显 tab:浅灰图标 + 灰色文字 + `cursor: not-allowed` 暗示(交互态不响应)
- Tab 高度 36px 圆角胶囊形,横向居中分布

## v3 实施:开始写代码

**用户决策**:
- Apple Silicon 独占,不兼容 Intel
- 代码风格简约干净、不冗余
- 需要交付 .dmg

**实施成果**:
- `TECH_DECISIONS.md`:锁定技术路线(嵌入 PBS / arm64 / Swift 5.9 / macOS 14+)
- SwiftUI 完整骨架(13 个文件,每个 ≤ 200 行,共 14 个 view)
  - 入口 + AppState(Observable) + ContentView + Module Router
  - TitleBarView / TrafficLights / ModuleTabBarView / TabPill
  - SiteCrawler 子模块:URL 输入 / 类型 Chip / 路径选择 / 停止+开始按钮 / 日志面板
  - Bridge:CrawlerProcess 子进程 + CrawlerMessages 协议类型
- Python 端完整骨架
  - `protocol.py` JSON 行 emit
  - `cli.py` stdio 命令分发
  - `crawler/spider.py` BFS 爬虫(httpx + selectolax + tldextract)
  - `crawler/url_frontier.py` 队列 + 去重
  - `downloader/fetcher.py` 流式下载 + 文件名清洗 + 去重命名
  - `config.py` 扩展名 + 文件夹映射
- `tests/test_smoke.py`:端到端 smoke test,起本地 HTTP server,验证 5 个资源下载 + 分类 **通过**
- `build/release.sh`:一键打 .dmg(下载 PBS / xcodebuild / 嵌入 Python / create-dmg)
- `SwiftApp/build/gen-xcode.sh`:xcodegen 工程生成

**Smoke test 状态**:✅ PASSED (1 passed in 0.81s)
